# Arquitectura - Manual de Sonido LAPEN

## Descripción general

La aplicación estará dividida en diferentes partes para facilitar su crecimiento y mantenimiento.

---

# Estructura principal

## Aplicación (Frontend)

Será la parte que ve el usuario.

Funciones:

- Pantallas de la aplicación
- Menús
- Formularios
- Material educativo
- Interacción con usuarios

---

## Servidor (Backend)

Será el encargado de manejar la información.

Funciones:

- Usuarios
- Permisos
- Eventos
- Datos de aprendizaje
- Comunicación con la base de datos

---

## Base de datos

Guardará la información de la aplicación.

Ejemplos:

- Usuarios registrados
- Cursos realizados
- Eventos creados
- Equipos utilizados
- Historial de actividades

---

# Futuro crecimiento

La arquitectura permitirá agregar nuevos módulos:

- Iluminación
- Video
- Streaming
- Electricidad
- Logística

---

# Objetivo técnico

Crear una aplicación organizada, escalable y fácil de mantener.
